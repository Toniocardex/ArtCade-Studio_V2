# Prototype Sprite ADR

**Status:** Accepted  
**Format:** `PROJECT_FORMAT_V4`

## Principle

Authoring stores stable `ImageAsset.id` in `spriteAssetId`. Runtime and WASM upload boundaries resolve to project-relative load keys via `resolveImageLoadKey` / `resolveSpriteLoadKey` only.

| Authoring state | Result |
|-----------------|--------|
| `spriteAssetId === null` | Intentional no visual |
| ID resolved, `source: generated` | Prototype sprite |
| ID resolved, imported/promoted | Normal sprite |
| ID not resolved | Broken reference |
| Load key present, file missing at runtime | Magenta missing texture |

## Normative rule

`spriteAssetId` must **never** be used directly as path, texture key, or upload key. Use `imageAssetForRef` + `resolveImageLoadKey` (or `resolveSpriteLoadKey` on `SpriteComponent`).

## Creation

Only on `OBJECT_TYPE_ADD`, atomic reducer pass:

```ts
type ObjectTypeAddAction = {
  type: 'OBJECT_TYPE_ADD'
  typeId: string
  displayName: string
  prototypeAsset: ImageAsset  // pre-materialized; reducer is pure
}
```

Call sites use `buildObjectTypeAddAction(displayName)` before dispatch.

## Generated asset model

- `spriteAssetId` = `asset.id`
- Virtual load key: `__generated__/prototype/<asset-id>.png`
- `ImageAsset.source: 'imported' | 'generated'`
- `generated`: `{ generator: 'prototype-sprite', width, height, temporary, shape, baseColor, ownerTypeId?, modified? }`
- `dataUrl`: RGBA consumed by editor/runtime upload

## Runtime

`runtime-fingerprint` / `projectJsonForRuntime` emit resolved load keys in sprite `a` field. C++ skips draw when load key empty; missing files use texture-cache magenta (no `fillColor` rectangle fallback).

## Save vs export

Save may keep embedded `dataUrl` + virtual path. Export must materialize generated bytes into the pack.

## Migration

V3 → V4: empty or path-only sprite refs on object types become generated prototype assets; legacy `fillColor` preserved on the type.

## Inspector actions

| Action | Effect |
|--------|--------|
| **Promote** | Writes `dataUrl` bytes to `assets/images/<id>_<name>.png`, sets `source: 'imported'`, drops `generated` metadata; keeps clips / image points / stable `id`. |
| **Reset prototype** | Regenerates factory rectangle (`colorFromObjectTypeId`), clears clips, collision profile, and clip fields on object types using the asset. |


| Module | Role |
|--------|------|
| `editor/src/utils/sprite-asset-ref.ts` | Canonical ref + load-key helpers |
| `editor/src/utils/prototype-sprite.ts` | Generator, `buildObjectTypeAddAction`, GC, migration |
| `editor/src/utils/resolve-image-load-key.ts` | Dual-read id/path resolver |
